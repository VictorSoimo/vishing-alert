# GuardianShield Prototype - 1 Month Development Workplan

## 🎯 Revised Prototype Goals (With Speakerphone Workaround)

**Primary Focus:** SMS Fraud Detection + REAL Vishing Detection via Speakerphone
- **SMS Detection:** Fully functional on-device NLP for smishing detection
- **Call Screening:** Implement CallScreeningService for caller identification
- **LIVE Vishing Detection:** Real-time call analysis when speakerphone is enabled
  - Automatic speakerphone activation option
  - Live audio recording via microphone
  - Real-time speech-to-text transcription
  - Fraud indicator detection during active calls
  - Visual alerts without interrupting call

**Why this approach:**
- Speakerphone audio CAN be captured via microphone (AudioSource.MIC)
- Demonstrates full end-to-end vishing protection capability
- Real, working feature (not just simulation)
- Privacy-preserving (all processing on-device)
- Honest about limitations (requires speakerphone)

  ```kotlin
  class CallAudioManager {
      fun enableSpeakerphone() {
          audioManager.mode = AudioManager.MODE_IN_CALL
          audioManager.isSpeakerphoneOn = true
          // Show persistent notification: "GuardianShield Active - Speakerphone Required"
      }
      
      fun startRecording(): AudioRecord {
          val bufferSize = AudioRecord.getMinBufferSize(
              16000, // 16kHz sample rate (good for speech)
              AudioFormat.CHANNEL_IN_MONO,
              AudioFormat.ENCODING_PCM_16BIT
          )
          
          return AudioRecord(
              MediaRecorder.AudioSource.MIC, // Captures speakerphone audio
              16000,
              AudioFormat.CHANNEL_IN_MONO,
              AudioFormat.ENCODING_PCM_16BIT,
              bufferSize
          ).apply { startRecording() }
      }
  }
  ```

  ```kotlin
  class VishingDetectionService : Service() {
      private lateinit var audioRecord: AudioRecord
      private var isRecording = false
      
      override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
          showForegroundNotification("Monitoring call for fraud...")
          startCallMonitoring()
          return START_STICKY
      }
      
      private fun startCallMonitoring() {
          isRecording = true
          audioRecord = callAudioManager.startRecording()
          
          // Process audio in chunks
          thread {
              val buffer = ShortArray(1024)
              while (isRecording) {
                  val read = audioRecord.read(buffer, 0, buffer.size)
                  if (read > 0) {
                      processAudioChunk(buffer)
                  }
              }
          }
      }
  }
  ```

**Days 21-22: Real-Time Speech-to-Text**
- [ ] Integrate Vosk for offline STT:
  ```kotlin
  class RealtimeTranscriber {
      private lateinit var model: Model
      private lateinit var recognizer: Recognizer
      private val transcriptBuffer = StringBuilder()
      
      fun initialize(context: Context) {
          // Extract model from assets (download vosk-model-small-en-us-0.15.zip)
          val modelPath = extractModel(context)
          model = Model(modelPath)
          recognizer = Recognizer(model, 16000.0f) // Match recording sample rate
          recognizer.setMaxAlternatives(0)
          recognizer.setWords(true)
      }
      
      fun processAudioChunk(audioData: ShortArray): String? {
          // Convert short array to byte array
          val bytes = ByteArray(audioData.size * 2)
          ByteBuffer.wrap(bytes).order(ByteOrder.LITTLE_ENDIAN)
              .asShortBuffer().put(audioData)
          
          if (recognizer.acceptWaveForm(bytes, bytes.size)) {
              val result = recognizer.result
              val text = parseResult(result)
              transcriptBuffer.append(text).append(" ")
              return text
          }
          return null
      }
      
      fun finalize(): String {
          val finalResult = recognizer.finalResult
          return transcriptBuffer.toString()
      }
  }
  ```

**Days 23-24: Real-Time Vishing Detection**
- [ ] Adapt SMS fraud detection for call transcripts with REAL-TIME analysis:
  ```kotlin
  class RealtimeVishingDetector {
      private val detectionWindow = 30 // seconds
      private val recentTranscript = CircularBuffer<String>(capacity = 100)
      private var fraudScore = 0f
      private val detectedIndicators = mutableSetOf<FraudIndicator>()
      
      fun analyzeTranscriptChunk(newText: String): DetectionResult {
          recentTranscript.add(newText)
          val contextWindow = recentTranscript.joinToString(" ")
          
          // Check for vishing indicators
          val impersonationScore = detectImpersonation(contextWindow)
          val pressureScore = detectPressureTactics(contextWindow)
          val infoRequestScore = detectSensitiveInfoRequests(contextWindow)
          val urgencyScore = detectUrgency(contextWindow)
          
          // Update rolling fraud score
          fraudScore = calculateVishingScore(
              impersonationScore,
              pressureScore,
              infoRequestScore,
              urgencyScore
          )
          
          // Trigger alert if threshold exceeded
          if (fraudScore > 70) {
              return DetectionResult(
                  alertLevel = AlertLevel.HIGH,
                  score = fraudScore,
                  indicators = detectedIndicators.toList(),
                  recommendation = "CAUTION: Possible fraud call. Do NOT share personal information."
              )
          }
          
          return DetectionResult.SAFE
      }
      
      private fun detectImpersonation(text: String): Float {
          val impersonators = listOf(
              "IRS", "FBI", "Social Security", "Medicare",
              "Microsoft", "Amazon", "Apple", "Google",
              "Bank of", "Wells Fargo", "Chase", "PayPal",
              "technical support", "customer service", "fraud department"
          )
          
          var score = 0f
          val lowerText = text.lowercase()
          impersonators.forEach { entity ->
              if (lowerText.contains(entity.lowercase())) {
                  score += 25f
                  detectedIndicators.add(FraudIndicator.IMPERSONATION)
              }
          }
          return score.coerceAtMost(100f)
      }
      
      private fun detectPressureTactics(text: String): Float {
          val pressureWords = listOf(
              "warrant", "arrest", "legal action", "lawsuit", "suspended",
              "compromised", "hacked", "unauthorized", "fraudulent",
              "immediate", "urgent", "right now", "within 24 hours",
              "last chance", "final notice", "consequences"
          )
          
          var score = 0f
          val lowerText = text.lowercase()
          pressureWords.forEach { word ->
              if (lowerText.contains(word)) {
                  score += 15f
                  detectedIndicators.add(FraudIndicator.PRESSURE_TACTICS)
              }
          }
          return score.coerceAtMost(100f)
      }
      
      private fun detectSensitiveInfoRequests(text: String): Float {
          val sensitiveRequests = listOf(
              "social security number", "SSN", "credit card", "card number",
              "CVV", "PIN", "password", "account number", "routing number",
              "date of birth", "mother's maiden name", "verify your identity",
              "confirm your", "provide your"
          )
          
          var score = 0f
          val lowerText = text.lowercase()
          sensitiveRequests.forEach { request ->
              if (lowerText.contains(request)) {
                  score += 30f // High weight for info requests
                  detectedIndicators.add(FraudIndicator.INFO_REQUEST)
              }
          }
          return score.coerceAtMost(100f)
      }
  }
  ```

## ⚠️ Known Limitations & Workarounds

### 1. **Speakerphone Requirement for Live Call Analysis**
- **Issue:** Android blocks direct call audio access for third-party apps
- **Workaround:** 
  - Capture audio via microphone when speakerphone is enabled
  - Both sides of conversation captured together
  - Requires user to enable speakerphone (automatic or manual)
  - **Trade-off:** Privacy reduced in public spaces, but fraud protection gained
  - **Mitigation:** Clear warnings, user consent, private space recommendations

### 2. **Audio Quality & Environment Noise**
- **Issue:** Microphone captures ambient noise along with call audio
- **Workaround:**
  - Use noise suppression in AudioRecord (AudioSource.VOICE_RECOGNITION)
  - Implement voice activity detection
  - Require quiet environment for best results
  - Provide audio quality indicator to user

### 3. **Battery Consumption**
- **Issue:** Continuous recording + STT + NLP is battery-intensive
- **Workaround:**
  - Optimize ML models for efficiency
  - Use lightweight STT model
  - Process audio in chunks, not continuously
  - Allow user to disable real-time transcription (analysis only)
  - Show battery usage prominently

### 4. **Accessibility API Ban**
- **Issue:** Google banned Accessibility API for call recording on Play Store
- **Workaround:** 
  - Don't use Accessibility APIs at all
  - Use legitimate CallScreeningService + speakerphone approach
  - Fully comply with Play Store policies
  - Be transparent in app description

### 5. **Model Size & Accuracy Trade-off**
- **Issue:** On-device models must be small (<10MB) for quick inference
- **Workaround:**
  - Use lightweight models (Vosk-small, TF-IDF)
  - Hybrid rule-based + ML approach
  - Focus on high-precision rather than high-recall
  - Allow model updates via app updates
